## Git Workflow

This project uses **GitHub Flow**:

1. `main` branch is always deployable
2. Create feature branches from `main` (e.g., `feature/s1-architecture`)
3. Open a Pull Request for review
4. Merge to `main` after validation
5. Use conventional commits for traceability (e.g., `feat:`, `fix:`, `test:`, `docs:`, `ci:`)
